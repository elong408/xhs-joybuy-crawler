# 小红书 Joybuy 每日内容采集与分析设计

日期：2026-06-11

## 目标

搭建一个本地运行的采集与分析流程，每天从已登录的小红书网页中检索 `joybuy` 相关内容，整理当日/近期头部笔记，并输出结构化数据与分析总结。

## 合规边界

1. 不绕过登录、验证码、风控、付费墙或平台访问限制。
2. 使用用户本人已登录的浏览器页面进行可见内容采集。
3. 低频采集，避免高并发、批量撞接口、模拟异常行为。
4. 如果出现验证码、登录过期、访问异常，脚本停止并提示用户手动处理。
5. 只采集用户在页面上可见或打开笔记后可见的信息。

## 采集关键词

默认关键词：

- `joybuy`
- `Joybuy`
- `京东 joybuy`
- `京东国际 joybuy`
- `JD joybuy`

关键词可在配置文件中增删。

## 采集字段

每条笔记尽量保存：

| 字段 | 说明 |
|---|---|
| crawl_date | 采集日期 |
| keyword | 搜索关键词 |
| rank | 搜索结果页中的顺序 |
| note_id | 笔记 ID |
| url | 笔记链接 |
| title | 标题 |
| author | 作者昵称 |
| publish_time | 发布时间/相对时间 |
| like_count | 点赞数 |
| comment_count | 评论数 |
| collect_count | 收藏数，若页面不可见则为空 |
| note_type | 图文/视频/未知 |
| content | 正文或可见摘要 |
| images_count | 图片数量，若可见 |
| raw_text | 页面可见原文备份 |

## 分析字段

在采集字段基础上生成：

| 字段 | 说明 |
|---|---|
| engagement_score | `点赞 + 评论*3 + 收藏*2`，缺失字段按 0 处理 |
| primary_category | 一级分类 |
| secondary_category | 二级分类 |
| sentiment | 正向/中性/负向 |
| risk_type | 品控/物流/售后/价格/舆情/无明显风险 |
| user_need | 用户核心诉求 |
| summary | 单条笔记摘要 |

## 分类规则

一级分类：

1. 购物体验：下单、物流、清关、售后、配送。
2. 价格促销：优惠、补贴、折扣、比价、大促。
3. 商品品类：美妆、母婴、保健品、数码、食品、生鲜等。
4. 真假信任：正品、授权、假货质疑、鉴别、避雷。
5. 海外业务：英国/欧洲/海外本地购物、配送、与 Amazon/Temu/本地商超对比。
6. 品牌资讯：Joybuy/JD 业务动态、平台介绍、活动公告。

情绪规则：

- 正向：推荐、好价、物流快、体验好、复购。
- 中性：提问、攻略、信息搬运、活动提醒。
- 负向：避雷、投诉、假货质疑、售后失败、物流延迟。

## 输出文件

按日期输出到 `output/YYYY-MM-DD/`：

1. `raw_notes.csv`：原始采集结果。
2. `analyzed_notes.csv`：带分类、情绪、分数的结果。
3. `daily_report.md`：每日分析总结。
4. `daily_report.html`：可浏览报告。
5. `screenshots/`：必要时保存采集页面截图，便于人工核验。

## 每日报告结构

1. 今日概览：采集关键词、采集时间、笔记数量、有效笔记数量。
2. 头部内容：按互动分数排序 Top 10。
3. 主题分布：各分类数量与占比。
4. 情绪分布：正/中/负数量与占比。
5. 风险摘要：负面与高风险内容列表。
6. 用户关注点：价格、物流、正品、售后、商品等高频诉求。
7. 运营建议：内容方向、风险应对、关键词建议。

## 技术方案

采用 Playwright 本地浏览器自动化：

1. 首次运行时打开持久化浏览器用户目录。
2. 用户手动登录小红书。
3. 脚本进入搜索页，输入关键词。
4. 慢速滚动结果页，读取可见卡片。
5. 对排名靠前的笔记逐条打开详情页，读取正文与互动数据。
6. 保存 CSV。
7. 运行本地规则分析，生成 Markdown/HTML 报告。

## 配置文件

`config/joybuy_crawler.json`：

```json
{
  "keywords": ["joybuy", "Joybuy", "京东 joybuy", "京东国际 joybuy", "JD joybuy"],
  "max_notes_per_keyword": 20,
  "open_detail_top_n": 10,
  "min_delay_seconds": 3,
  "max_delay_seconds": 8,
  "headless": false
}
```

## 成功标准

1. 能在用户已登录状态下采集至少一个关键词的可见搜索结果。
2. 能输出包含标题、作者、链接、互动数据的 CSV。
3. 能生成每日 Top、分类、情绪、风险总结。
4. 遇到登录/验证码/页面异常时不会绕过，而是停止并提示人工处理。

## 非目标

1. 不做账号池、代理池、绕验证码、绕签名接口。
2. 不承诺采集平台未展示或不可访问的数据。
3. 不做大规模高频爬取。
